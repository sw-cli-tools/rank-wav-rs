How could I create a Rust-based CLI that can scan a specified dir and rank the wav files the way you just did?


Thought for 17s
You want a Rust CLI that:

scans a directory for .wav files,

computes a few perceptual-ish audio features,

ranks files by rules like the ones I just used, and

prints a sortable report.

That is a very sane little bat-cave utility. The clean architecture is:

High-level design
Pipeline

Discover files in a directory

Decode WAV into normalized mono samples

Extract features

RMS energy

zero-crossing rate

spectral centroid

spectral bandwidth

Score each file with one or more ranking formulas

Output as table / CSV / JSON

For a WAV-only first version, hound is a good fit because it is specifically for reading and writing WAV files, and its WavReader is streaming-oriented. For FFT work, rustfft is a widely used pure-Rust FFT library with SIMD support on supported x86_64 machines. If you later want MP3/FLAC/M4A too, symphonia is the natural upgrade path because it is a pure-Rust decoding/demuxing framework for many audio formats. 

Nonfunctional requirements
You’ve got a few tradeoffs:

Fast enough
For short motifs, even a naive implementation will be quick. rustfft is more than sufficient here. 

Deterministic
You want ranking formulas that are explicit and stable, not mystical goblin vibes.

Extensible
Today: WAV files.
Later: FLAC/MP3/AAC via symphonia, extra metrics, maybe batch CSV export.

Explainable
Every score should be decomposable into the metrics that produced it. Otherwise the CLI becomes a fake oracle in a trench coat.

Feature set to implement
I’d start with exactly these metrics:

1. RMS
Measures signal strength / loudness-ish presence.

2. Zero-crossing rate
A rough proxy for noisiness or edge harshness.

3. Spectral centroid
A rough proxy for brightness.

4. Spectral bandwidth
How spread-out the spectrum is; often correlates with roughness/complexity.

These are simple enough to compute and line up reasonably well with the kind of ranking we discussed.

Ranking logic
Use two scores, because “best” and “most pleasing” are not the same beast.

“Most pleasing”
Bias toward:

lower centroid

lower bandwidth

lower ZCR

moderate RMS

A simple formula:

pleasing_score =
    -0.40 * norm(centroid)
    -0.30 * norm(bandwidth)
    -0.20 * norm(zcr)
    +0.10 * norm(rms)
“Best”
Bias toward:

stronger RMS

moderate centroid

moderate bandwidth

not too noisy

A simple formula:

best_score =
    +0.35 * norm(rms)
    -0.25 * distance_from_target(norm(centroid), 0.45)
    -0.20 * distance_from_target(norm(bandwidth), 0.40)
    -0.20 * norm(zcr)
That second formula treats “best” as balanced and present, rather than merely dark and soft.

use anyhow::Result;
use clap::Parser;
use cli::{Cli, SortMode};

fn main() -> Result<()> {
    let cli = Cli::parse();

    let mut rows = scan::scan_dir(&cli.dir, cli.recursive)?;

    score::normalize_rows(&mut rows);
    score::compute_scores(&mut rows);

    match cli.sort {
        SortMode::Pleasing => {
            rows.sort_by(|a, b| b.pleasing_score.total_cmp(&a.pleasing_score));
        }
        SortMode::Best => {
            rows.sort_by(|a, b| b.best_score.total_cmp(&a.best_score));
        }
    }

    output::print_rows(&rows, cli.sort, cli.json)?;
    Ok(())
}

src/scan.rs
use crate::features::{compute_features, FeatureRow};
use crate::wav::read_wav_mono_f32;
use anyhow::Result;
use std::path::Path;
use walkdir::WalkDir;

pub fn scan_dir(dir: &Path, recursive: bool) -> Result<Vec<FeatureRow>> {
    let max_depth = if recursive { usize::MAX } else { 1 };
    let mut rows = Vec::new();

    for entry in WalkDir::new(dir).max_depth(max_depth) {
        let entry = entry?;
        let path = entry.path();

        if !entry.file_type().is_file() {
            continue;
        }

        if path.extension().and_then(|s| s.to_str()).map(|s| s.eq_ignore_ascii_case("wav")) != Some(true) {
            continue;
        }

        match read_wav_mono_f32(path) {
            Ok((samples, sr)) => {
                let mut row = compute_features(path, &samples, sr)?;
                row.sample_rate = sr;
                row.num_samples = samples.len();
                rows.push(row);
            }
            Err(err) => {
                eprintln!("Skipping {}: {err}", path.display());
            }
        }
    }

    Ok(rows)
}
src/wav.rs
use anyhow::{bail, Result};
use hound::{SampleFormat, WavReader};
use std::path::Path;

pub fn read_wav_mono_f32(path: &Path) -> Result<(Vec<f32>, u32)> {
    let mut reader = WavReader::open(path)?;
    let spec = reader.spec();

    let channels = spec.channels as usize;
    let sample_rate = spec.sample_rate;

    if channels == 0 {
        bail!("wav has zero channels");
    }

    let interleaved: Vec<f32> = match (spec.sample_format, spec.bits_per_sample) {
        (SampleFormat::Int, 8) => reader
            .samples::<i8>()
            .map(|s| s.map(|v| v as f32 / i8::MAX as f32))
            .collect::<Result<Vec<_>, _>>()?,

        (SampleFormat::Int, 16) => reader
            .samples::<i16>()
            .map(|s| s.map(|v| v as f32 / i16::MAX as f32))
            .collect::<Result<Vec<_>, _>>()?,

        (SampleFormat::Int, 24 | 32) => reader
            .samples::<i32>()
            .map(|s| s.map(|v| v as f32 / i32::MAX as f32))
            .collect::<Result<Vec<_>, _>>()?,

        (SampleFormat::Float, 32) => reader
            .samples::<f32>()
            .collect::<Result<Vec<_>, _>>()?,

        _ => bail!(
            "unsupported WAV format: {:?} {} bits",
            spec.sample_format,
            spec.bits_per_sample
        ),
    };

    if channels == 1 {
        return Ok((interleaved, sample_rate));
    }

    // Downmix to mono by averaging channels.
    let mut mono = Vec::with_capacity(interleaved.len() / channels);
    for frame in interleaved.chunks_exact(channels) {
        let sum: f32 = frame.iter().copied().sum();
        mono.push(sum / channels as f32);
    }

    Ok((mono, sample_rate))
}
src/features.rs
use anyhow::{bail, Result};
use rustfft::{num_complex::Complex, FftPlanner};
use serde::Serialize;
use std::f32::consts::PI;
use std::path::{Path, PathBuf};

#[derive(Debug, Clone, Serialize)]
pub struct FeatureRow {
    pub path: PathBuf,
    pub sample_rate: u32,
    pub num_samples: usize,

    pub rms: f32,
    pub zcr: f32,
    pub spectral_centroid: f32,
    pub spectral_bandwidth: f32,

    pub rms_norm: f32,
    pub zcr_norm: f32,
    pub centroid_norm: f32,
    pub bandwidth_norm: f32,

    pub pleasing_score: f32,
    pub best_score: f32,
}

pub fn compute_features(path: &Path, samples: &[f32], sample_rate: u32) -> Result<FeatureRow> {
    if samples.len() < 128 {
        bail!("too few samples");
    }

    let rms = compute_rms(samples);
    let zcr = compute_zcr(samples);
    let (spectral_centroid, spectral_bandwidth) = compute_spectral_features(samples, sample_rate);

    Ok(FeatureRow {
        path: path.to_path_buf(),
        sample_rate,
        num_samples: samples.len(),
        rms,
        zcr,
        spectral_centroid,
        spectral_bandwidth,
        rms_norm: 0.0,
        zcr_norm: 0.0,
        centroid_norm: 0.0,
        bandwidth_norm: 0.0,
        pleasing_score: 0.0,
        best_score: 0.0,
    })
}

fn compute_rms(samples: &[f32]) -> f32 {
    let sum_sq: f32 = samples.iter().map(|x| x * x).sum();
    (sum_sq / samples.len() as f32).sqrt()
}

fn compute_zcr(samples: &[f32]) -> f32 {
    let mut crossings = 0usize;
    for w in samples.windows(2) {
        let a = w[0];
        let b = w[1];
        if (a >= 0.0 && b < 0.0) || (a < 0.0 && b >= 0.0) {
            crossings += 1;
        }
    }
    crossings as f32 / (samples.len().saturating_sub(1) as f32)
}

fn compute_spectral_features(samples: &[f32], sample_rate: u32) -> (f32, f32) {
    let n = samples.len().next_power_of_two().min(16384).max(1024);
    let start = if samples.len() > n { (samples.len() - n) / 2 } else { 0 };
    let slice = &samples[start..start + n.min(samples.len())];

    let mut buffer = vec![Complex { re: 0.0f32, im: 0.0f32 }; n];

    for (i, b) in buffer.iter_mut().enumerate().take(slice.len()) {
        let w = 0.5 - 0.5 * (2.0 * PI * i as f32 / (slice.len() as f32)).cos(); // Hann-ish
        b.re = slice[i] * w;
    }

    let mut planner = FftPlanner::<f32>::new();
    let fft = planner.plan_fft_forward(n);
    fft.process(&mut buffer);

    let half = n / 2;
    let bin_hz = sample_rate as f32 / n as f32;

    let mags: Vec<f32> = buffer[..half].iter().map(|c| c.norm_sqr().sqrt()).collect();

    let mag_sum: f32 = mags.iter().sum();
    if mag_sum <= 1e-12 {
        return (0.0, 0.0);
    }

    let centroid = mags
        .iter()
        .enumerate()
        .map(|(i, m)| i as f32 * bin_hz * *m)
        .sum::<f32>()
        / mag_sum;

    let bandwidth = (
        mags.iter()
            .enumerate()
            .map(|(i, m)| {
                let f = i as f32 * bin_hz;
                let d = f - centroid;
                d * d * *m
            })
            .sum::<f32>()
            / mag_sum
    )
        .sqrt();

    (centroid, bandwidth)
}
src/score.rs
use crate::features::FeatureRow;

pub fn normalize_rows(rows: &mut [FeatureRow]) {
    normalize_field(rows, |r| r.rms, |r, v| r.rms_norm = v);
    normalize_field(rows, |r| r.zcr, |r, v| r.zcr_norm = v);
    normalize_field(rows, |r| r.spectral_centroid, |r, v| r.centroid_norm = v);
    normalize_field(rows, |r| r.spectral_bandwidth, |r, v| r.bandwidth_norm = v);
}

fn normalize_field(
    rows: &mut [FeatureRow],
    get: impl Fn(&FeatureRow) -> f32,
    mut set: impl FnMut(&mut FeatureRow, f32),
) {
    let min = rows.iter().map(&get).fold(f32::INFINITY, f32::min);
    let max = rows.iter().map(&get).fold(f32::NEG_INFINITY, f32::max);

    let span = (max - min).max(1e-9);
    for row in rows {
        let v = (get(row) - min) / span;
        set(row, v);
    }
}

pub fn compute_scores(rows: &mut [FeatureRow]) {
    for row in rows {
        row.pleasing_score =
            -0.40 * row.centroid_norm
            -0.30 * row.bandwidth_norm
            -0.20 * row.zcr_norm
            +0.10 * row.rms_norm;

        row.best_score =
            +0.35 * row.rms_norm
            -0.25 * dist(row.centroid_norm, 0.45)
            -0.20 * dist(row.bandwidth_norm, 0.40)
            -0.20 * row.zcr_norm;
    }
}

fn dist(x: f32, target: f32) -> f32 {
    (x - target).abs()
}

use crate::cli::SortMode;
use crate::features::FeatureRow;
use anyhow::Result;
use tabled::{Table, Tabled};

#[derive(Tabled)]
struct PrintRow {
    file: String,
    rms: String,
    zcr: String,
    centroid: String,
    bandwidth: String,
    pleasing: String,
    best: String,
}

pub fn print_rows(rows: &[FeatureRow], _sort: SortMode, json: bool) -> Result<()> {
    if json {
        println!("{}", serde_json::to_string_pretty(rows)?);
        return Ok(());
    }

    let print_rows: Vec<PrintRow> = rows
        .iter()
        .map(|r| PrintRow {
            file: r.path.file_name().unwrap_or_default().to_string_lossy().to_string(),
            rms: format!("{:.4}", r.rms),
            zcr: format!("{:.4}", r.zcr),
            centroid: format!("{:.0}", r.spectral_centroid),
            bandwidth: format!("{:.0}", r.spectral_bandwidth),
            pleasing: format!("{:.3}", r.pleasing_score),
            best: format!("{:.3}", r.best_score),
        })
        .collect();

    println!("{}", Table::new(print_rows));
    Ok(())
}

Methodology notes
A few engineering cautions:

Normalize across the batch
You want rankings relative to the set being compared. A bright sound in one directory may be dark in another.

Downmix to mono
For ranking timbre-like properties, mono averaging is fine for v1.

Use a window before FFT
Without a window, spectral leakage muddies the centroid/bandwidth estimates. Even a simple Hann window helps.

Analyze a stable chunk
For very long files, you may want the center segment or multiple segments averaged together instead of the whole file.

Pros and cons of this approach
Benefits

Risks / limitations
“Pleasing” is subjective and culturally slippery

RMS is not perceptual loudness

Centroid/bandwidth are coarse proxies, not psychoacoustic truth

So this is a useful heuristic tool, not a machine that has discovered beauty itself. That road leads to incense, velvet capes, and trouble.
